3. Génération des paires d'entraînement

3.1 Stratégie de génération

Pour le fine-tuning avec MultipleNegativesRankingLoss, chaque paire d'entraînement est de la forme (query, positive_passage). GPT-4o-mini génère 3 questions différentes par article (factuelle, pratique, conditionnelle) pour maximiser la diversité.

Prompt template :
"Voici un article de loi français. Génère 3 questions DIFFÉRENTES :
- une factuelle (définition, principe)
- une pratique (cas concret, que faire si...)
- une sur les conditions/exceptions

Article : {article} ({code})
Texte : {texte[:1000]}

Retourne JSON : {\"questions\": [\"q1\", \"q2\", \"q3\"]}"

4.2 Fonction de perte — MultipleNegativesRankingLoss

La fonction MultipleNegativesRankingLoss (Henderson et al., 2017) est la référence pour le fine-tuning de bi-encodeurs en information retrieval. Dans un batch de taille N :

  • Chaque paire (query_i, positive_i) constitue un exemple positif
  • Les (N-1) autres positifs du batch deviennent automatiquement des négatifs in-batch
  • Le modèle apprend à maximiser la similarité cosinus des paires positives
  • Avec batch_size=128 : chaque query est comparée à 127 négatifs — très efficace
from sentence_transformers import SentenceTransformer, InputExample
from sentence_transformers.losses import MultipleNegativesRankingLoss

model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")
loss  = MultipleNegativesRankingLoss(model)

# Format InputExample(texts=[query, positive])
train_examples = [InputExample(texts=[p["query"], p["positive"]]) for p in paires]

model.fit(
    train_objectives=[(dataloader, loss)],
    epochs=25,
    warmup_steps=200,
    evaluator=evaluator,          # InformationRetrievalEvaluator
    evaluation_steps=500,
    save_best_model=True,         # checkpoint epoch 15 (meilleur NDCG@10)
    output_path="lexai-embeddings"
)

7.4 Analyse et interprétation

Pourquoi le fine-tuning améliore le retrieval (+20%) mais pas les métriques RAGAS ?

Les métriques IR (Hit Rate, NDCG) mesurent si le bon article est dans les Top-K résultats. Les métriques RAGAS mesurent la qualité de la réponse finale générée par GPT-4o-mini.

GPT-4o-mini est un modèle très robuste : même avec des articles légèrement moins pertinents, il génère une réponse correcte et non-hallucinée. Cela crée un ceiling effect — GPT "absorbe" les imperfections du retrieval, masquant le gain du fine-tuning dans les métriques RAGAS.

Conclusion : le fine-tuning apporte un gain réel et mesurable sur le retrieval. Le gain ne se répercute pas visiblement sur RAGAS car le LLM génération est suffisamment fort pour compenser. Les deux approches sont équivalentes en qualité de réponse finale (écart < 3%).
